## Findings considered and rejected

Recorded so nobody re-audits them. Each was opened and confirmed by the advisor.

- **Weighted-pick last-row bias** (`modules/packs/pick.ts`): the `roll >= totalWeight` fallback to the last row is standard weighted-selection and documented as an accepted float-drift edge; bias is asymptotically zero. Not a bug.
- **Stock decrement race / negative stock** (`workflows/steps/decrement-card-stock.ts`): explicitly by-design — "STOCK IS A COUNTER, NOT A GATE." The adjust is guarded by `target.stocked > 0`, and the one-unit drift is documented as acceptable. Not a bug.
- **Notification mark-read TOCTOU / duplicate rows** (`store/notifications/[id]/read`): a partial unique index on `(notification_id, customer_id) WHERE deleted_at IS NULL` already exists (`models/notification-read.ts`); the retry-on-conflict re-fetch is correct. Only a rare double-miss re-throw remains (cosmetic).
- **CSRF on state-changing store POSTs** (open, buyback, topup, draw, delivery, referral): customer auth is **bearer-only** — the backend issues no customer session cookie, so there is no ambient-credential CSRF vector. Documented in `middlewares.ts:32-44`. Not a bug.
- **Admin double-submit on money buttons**: `@medusajs/ui` `Button` with `isLoading` renders as `disabled`, and the handlers (`requestAdjust`/`applyAdjust`) early-return while `isPending`. Guarded.
- **Slot `settled` ref "never cleared"** (`SlotReelColumn.tsx`): the main effect sets `settled.current = false` on its first line, and the column remounts per spin (keyed by `spinKey`). Reset is correct.
- **Slot `onSettled` stale closure** (`SlotReelStack.tsx`): `onSettledRef` is refreshed every render via an effect; the callback always reads the current value. Correct pattern.
- **`availableBalance` staleness / SHA-256 idempotency collision / soft-delete replay**: all defended by the per-customer advisory lock in `mutateCreditAtomic` and existing idempotency tests. Not bugs.
- **Media upload path traversal** (`admin/media/route.ts`): `path.basename()` after `\`→`/` normalization is sufficient; multer memory storage + `validateImage` (mime/magic-byte/size/dims) gate the rest.
- **"Verify topup/draw rate limit + CSRF exists"** (raised by the money-path audit): confirmed present in `middlewares.ts` — `createCreditTopupRateLimit()` on `/store/credits/topup`, `deliveryWriteRateLimit` on `/store/rewards/*`, all after `authenticate('customer', ['bearer'])`.

Generated by the `improve` skill (deep tier) against commit `e9ce6968`
(branch `feat/virtual-shop`; working tree dirty at planning time — small
uncommitted changes to the from-pricecharting flow, `SlotMachineClient.tsx`,
sim scripts, `package.json`). Method: eight parallel read-only audits —
backend money internals (the round-2-unread commission/VIP/reward-box/jobs

and direction. Every table-bound finding was re-opened and confirmed by the
advisor against the live code. Same caveat as round 2: **static code-read, no
runtime exercise.**

Run constraint honored: several agent worktrees were active during this run
(plans 009–018 executors, `perf/p1-frames-listcache`, `pixelslot-rebrand`,
sim branches). Round-3 plans avoid every file those touch; where a plan edits
`service.ts` (shared with TODO plan 011) the plan carries an explicit
rebase/drift instruction. Frames-list caching was excluded from the perf
audit entirely (in progress in a worktree).

**Headline.** The core held up again — and so did the layers above it. The
admin-API audit came back _clean_ (middleware coverage map verified against
Medusa framework source and Express 4 path-to-regexp wildcard semantics; every
admin/store write validates; public feeds mask PII). The money internals
audit's only finding is an availability issue, not a correctness one. The
real round-3 story is operational and trust debt: **CI never type-checks the
largest backend package; the flagship E2E money-loop spec has been disabled
since 2026-07-07; the hourly commission job holds every beneficiary's credit
lock in one transaction; the public profile route loads 20k rows per view;
and several publicly reachable pages present fabricated data — including
"verified" badges on real companies — as live product.**

### Round-3 Direction (options for the maintainer — not bugs, coarse estimates)

1. **P2P selling/trading — the missing verb.** `PRODUCT.md:18` promises
   "hold, trade, redeem, or sell back"; `how-it-works/page.tsx:142` promises
   "list it on the marketplace at your own price … trade cards with each
   other directly". No listing/trade/offer/transfer route or entity exists;
   the marketplace grid reads a single house seller (Mercur multi-vendor
   installed but unused); `/social` is the mock UI for it. L, HIGH risk
   (escrow/fraud/disputes) — scope as a listing-sale-first design spike, and
   note it only pays off after real cash-out exists.
2. **PSP integration + real cash-out readiness spike.** The code anticipates
   it by name: `topup.ts` "Batch B" seams, `topup-credits.ts` "the real
   gateway slots in here" with the idempotency contract already threaded;
   `/bank-withdrawal` says payouts go live with the gateway. The compliance
   half (AMLA/KYC) is the long pole — enumerate blockers before building.
3. **Launch the dormant rewards economy.** `REWARDS_REDEMPTION_ENABLED`
   default-off ("dormant until Phase P") gates already-built claim/draw/
   withdraw routes, admin editors, and storefront surfaces. A launch spike =
   enumerate remaining Phase-P work (economy parameters, abuse limits,
   runbook), then staged rollout. M-ish, MED risk.
4. **Promote the sim harness to a nightly economy-regression gate.** The
   2-day pilot caught a P1 money bug + 10 others with 0 harness errors;
   `ledger.mjs` invariants exist; nothing in CI references sim. M, LOW risk —
   and it's the safety net options 1–3 all want.
5. **Mock-route triage (DIR-05).** Keep-as-static vs delete per route:
   `roulette` (second gacha mechanic — delete unless wanted), `merchants` /
   `pokemon` (static reference or delete), `social`/`activity`/`repacks`
   (fold into the trading track — don't delete), honest placeholders
   (`30th`, `airdrop`, `free`, `clawmaker`, `bank-withdrawal`) keep. Plan 024
   makes everything honest in the interim, so this decision loses urgency.

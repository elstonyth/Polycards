Shared `image` params: `--aspect_ratio 1:1|16:9|9:16|4:3|3:4`, `--resolution 1K|2K|4K`, `--style` (`Photorealistic`, `Illustration`, `3D Render`, … 15+), `--number_of_images N`, `--output_format png|jpeg`, `--files` / `--file_urls` / `--ref_histories <uuid>` (plural — reuse previous generations).

**Accepted aspect ratios are a FIXED enum — the model can't emit anything else.** nano (`image`): `1:1 16:9 9:16 4:3 3:4` (portrait: 9:16≈0.563, 3:4=0.75). gpt-image: `1:1 16:9 9:16 4:3 3:4 21:9 3:2 2:3` (portrait: 9:16≈0.563, 2:3≈0.667, 3:4=0.75). grok-image/meta-image take `--orientation landscape|portrait|square` instead of a ratio. So the closest-to-portrait options are 9:16 (0.563) and 2:3 (0.667). **Nothing hits a graded card slab's ~0.6** — a slab-fitting frame must be built procedurally to the measured target, not generated-and-squeezed (see SKILL.md → "Measure the target BEFORE you generate" and workflows.md).

**Backend split that matters:** `nano-banana-*` = Google Gemini backend (throttles under load with `RESOURCE_EXHAUSTED`). `gpt-image` = separate backend, stays up. `grok-image` = xAI. When Gemini is congested and you need a reference-led image now, fall back to `gpt-image` (pricier).

## Picking flow

Match by intent; the first entry that fits wins.

### Image

1. **User names a model** → use it. Don't invent model names — if the API rejects one as unknown, check https://docs.snapgen.ai, don't guess variants.
2. **A reference exists** ("like this", "our logo", "variant of", "edit this") → `image --model nano-banana-2 --files ref.png`. grok/meta accept refs per the docs but are unproven — don't route reference work there by default. Iterating on a previous SnapGen result → reference it by uuid instead of re-uploading: `--ref_history` (gpt/grok/meta/sora), `--ref_histories` (plural, nano `image`).
3. **Critical in-image text/typography** (wordmark, label, UI copy) → `nano-banana-pro`, or `gpt-image` if Gemini is throttled or it's a final asset.
4. **Hero/final asset** (shipping to `public/images/`) → `gpt-image --mode high --resolution 2K+` — draft the direction cheaply first.
5. **Gemini throttled (`RESOURCE_EXHAUSTED`) and it can't wait** → `gpt-image` (medium/1K for drafts).
6. **Throwaway concept, cheapest possible** → `grok-image` (`--mode SPEED`).
7. **Everything else** → `image --model nano-banana-2`.

### Video

1. **User names a model/family** → use it.
2. **Image-reference-driven video** (animate this asset) → `video seedance --ref_images …` (best motion quality). Veo/kling also take `--ref_images` (local or URL); grok takes `--files`. Start/end-frame transition → `video veo --ref_images first.png,last.png --mode_image frame`.
   2b. **Continue/extend an existing clip** → `extend <family> "prompt" --ref_history <uuid of the original job>`.
3. **Cheap short clip** → `video kling --model kling-video-2-5 --mode relax` (cheapest) or `kling-video-3-0 --mode standard`.
4. **Needs voice/audio** → `kling-video-2-6 --mode professional_audio`.
5. **Cinematic/production quality** → `video veo --model veo-3.1` or `video sora --model sora-2-pro`.
6. **Default** → `video kling --model kling-video-3-0`.

Draft cheap, render the winner high-res, one job at a time.

## Higgsfield parity

**Doable here — use `references/playbooks.md`:** product photoshoots, marketplace listing cards, ads/UGC (Marketing Studio-style), and character-consistent sets (Soul-style, partial). Those Higgsfield features are private prompt enhancers on the same base models SnapGen exposes; the playbooks carry the equivalent templates.

**Truly unavailable (missing modality, no endpoint):** 3D/GLB, music/SFX, TTS/speech (docs intro mentions it but the public `openapi.json` has no audio endpoints), video analysis (Virality Predictor), generative `draw_to_video`/`reframe` (closest: `extend`/`storyboard`, or a local ffmpeg crop for reframe), and website building. There is also **no pre-submit cost endpoint** (Higgsfield's `generate cost`): validate with `--dry-run`, budget from the cost columns above, and read the `estimated_credit` the script prints at submit — failed jobs cost 0.

## Discovery

- `account` — plan + available/locked credits.
- `history [--filter_by image|video]` (default `all`) — recent jobs (uuid, status, model, prompt); any `--key value` passes as a query param (`--page`, `--items_per_page`).
- `status <uuid>` — full job JSON (params, result URLs, error).
- `--dry-run` — print the request without sending.
- Unwrapped housekeeping: `DELETE /uapi/v1/histories` (by ids/status/media_type) and `/histories/all[/{type}]` — soft-deletes, only records past the backend time limit; curl if ever needed.

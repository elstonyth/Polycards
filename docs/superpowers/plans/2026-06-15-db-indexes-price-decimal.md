# DB Indexes + Pack Price Decimal Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add five hot-column indexes to the `packs` module and convert `pack.price` from integer to decimal (`bigNumber`), so per-customer / feed / leaderboard reads stay off full-table scans as data grows and `$X.99` pack prices stop truncating.

**Architecture:** Model-declarative. Indexes are declared via Medusa's `.indexes()` DSL on three model files so `.snapshot-packs.json` stays the source of truth (no migration drift). `pack.price` flips to `model.bigNumber()`. Migrations are emitted by `medusa db:generate packs`; the price migration is then hand-edited to backfill the `raw_price` jsonb sidecar for the 8 existing rows. Two separate migrations (indexes, then price) for a clean, independently-revertible history.

**Tech Stack:** Medusa v2 module DML (`@medusajs/framework/utils`), MikroORM migrations, Postgres 16 (`pokenic-postgres` Docker, db `medusa`). Backend package manager: `corepack yarn` from `backend/packages/api`.

**Reference spec:** `docs/superpowers/specs/2026-06-15-db-indexes-price-decimal-design.md`

## Task 1: Declare the five indexes on the models

**Files:**
- Modify: `backend/packages/api/src/modules/packs/models/credit-transaction.ts`

Explicit `name:` on each index keeps the generated index name deterministic so the later EXPLAIN verification can assert it. All partial on `deleted_at IS NULL` (Medusa appends that predicate to every default query — confirmed via EXPLAIN).

- [ ] **Step 1: Add the index to `credit-transaction.ts`**

Change the `export const CreditTransaction = model.define(...)` so the `.define({...})` call is followed by `.indexes([...])`:

```ts
export const CreditTransaction = model
  .define("credit_transaction", {
    id: model.id().primaryKey(),
    customer_id: model.text(),
    // USD decimal (never cents). Positive = credit, negative = spend.
    amount: model.bigNumber(),
    reason: model.enum(["buyback", "topup", "pack_open", "adjustment"]),
    pull_id: model.text().unique().nullable(),
    reference: model.text().nullable(),
  })
  // Balance Σ + credits feed + admin gacha all read by customer_id ordered by
  // created_at; composite serves the filter + ORDER BY (+ pagination) in one scan.
  .indexes([
    {
      name: "IDX_credit_transaction_customer_id_created_at",
      on: ["customer_id", "created_at"],
      where: "deleted_at IS NULL",
    },
  ]);
```

(Keep the existing file comments above the model; only the `model.define(...)` → `model.define(...).indexes([...])` wrapping and the two new comment lines are added.)

- [ ] **Step 1: Change the field in `pack.ts`**

Replace `price: model.number(),` with:

```ts

  // column (+ raw_price jsonb sidecar) like every other money field here
  // (card.market_value, pull.buyback_amount, credit_transaction.amount).
  // model.number() mapped to integer and truncated any cents.
  price: model.bigNumber(),
```

- [ ] **Step 2: Typecheck**

Run: `corepack yarn build`
Expected: exit 0. `Number(pack.price)` reads at the call sites (leaderboard, charge step, store routes) compile unchanged — `bigNumber` reads as a string/number-coercible value, same as before.

- [ ] **Step 3: Generate the price migration**

Run: `corepack yarn medusa db:generate packs`
Expected: a second new `Migration<timestamp>.ts` appears; `.snapshot-packs.json` updates `pack.price` to `numeric` and adds `raw_price`.

- [ ] **Step 4: Replace the generated migration body with the hand-ordered version**

The generator emits DDL only and may add `raw_price NOT NULL` before any backfill, which fails on the 8 existing rows. Overwrite the new file's `up()`/`down()` with exactly this (keep the generated class name / timestamp):

```ts
import { Migration } from "@mikro-orm/migrations";

export class Migration<tsB> extends Migration {
  override async up(): Promise<void> {
    // int -> numeric so prices can hold cents (forward-compat; current data is
    // clean whole dollars). Ordered: retype, add nullable sidecar, backfill the
    // existing rows, then enforce NOT NULL. bigNumber stores value + precision
    // (matches the live raw_amount shape: {"value":"25","precision":20}).
    this.addSql(`ALTER TABLE "pack" ALTER COLUMN "price" TYPE numeric USING ("price"::numeric);`);
    this.addSql(`ALTER TABLE "pack" ADD COLUMN IF NOT EXISTS "raw_price" jsonb;`);
    this.addSql(`UPDATE "pack" SET "raw_price" = jsonb_build_object('value', "price"::text, 'precision', 20) WHERE "raw_price" IS NULL;`);
    this.addSql(`ALTER TABLE "pack" ALTER COLUMN "raw_price" SET NOT NULL;`);
  }

  override async down(): Promise<void> {
    // Lossy by nature: integer truncates any cents written while decimal.
    this.addSql(`ALTER TABLE "pack" DROP COLUMN IF EXISTS "raw_price";`);
    this.addSql(`ALTER TABLE "pack" ALTER COLUMN "price" TYPE integer USING (round("price")::integer);`);
  }
}
```

If the generated index-migration from Task 2 was somehow re-emitted here, delete those duplicate `CREATE INDEX` lines — this migration touches only `pack`.

- [ ] **Step 5: Apply the migration**

Run: `corepack yarn medusa db:migrate`
Expected: applied, no errors.

Expected: `\d pack` shows `price | numeric` and `raw_price | jsonb | not null`. The SELECT shows every row's `price` unchanged (25, 5000, 10, …) and `raw_price` populated like `{"value": "25", "precision": 20}` for all 8 rows.

- [ ] **Step 7: Prove decimals no longer truncate (non-destructive)**

Expected: returns `pokemon-rookie | 4.99` (then rolls back — no data change). Before the migration this stored `4`.
